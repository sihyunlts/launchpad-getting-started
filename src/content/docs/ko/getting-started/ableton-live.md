## Ableton Live 개요

Ableton Live는 런치패드를 제어하는 역할을 담당합니다. 모든 소리의 재생과 조명 효과의 처리가 소프트웨어 내부에서 이루어지므로, 기본적인 인터페이스 구성을 파악하는 것이 중요합니다.

### 인터페이스 주요 구성

Ableton Live의 메인 화면은 크게 네 가지 영역으로 구분됩니다.

- **상단부 (Control Bar):** 재생, 템포(BPM), 프로젝트 상태 등을 제어합니다.
- **좌측부 (Browser):** 오디오 클립, 플러그인, 가상 악기 등을 탐색하고 불러옵니다.
- **중앙부 (Session/Arrangement View):** 실제 음악 작업과 연주가 이루어지는 영역입니다.
- **하단부 (Detail View/Rack):** 선택된 트랙에 포함된 장치(악기, 이펙트 등)의 세부 설정을 관리합니다.

### 주요 시스템 제어 도구

- **BPM (Beats Per Minute):** 프로젝트의 전체 속도(템포)를 정의합니다.
- **메트로놈 (Metronome):** 정확한 박자 유지를 위한 안내음을 출력합니다.
- **재생(Play) / 정지(Stop) / 녹음(Record):** 트랙을 재생하거나, 멈추거나, 녹음할 수 있습니다.
- **MIDI 매핑 (MIDI Map):** 소프트웨어의 특정 기능을 런치패드 버튼에 직접 할당할 수 있는 기능입니다. 페이지 전환 등을 구현할 때 사용됩니다.
- **CPU 사용량:** CPU의 사용량을 실시간으로 모니터링합니다. 사용량이 높을 경우 오디오 끊김 현상이 발생할 수 있습니다.
- **화면 전환 (Tab):** Session 뷰와 Arrangement 뷰 사이를 전환합니다.

### 트랙(Track)의 구조와 역할

Ableton Live의 작업 영역은 수직 또는 수평의 '트랙'들로 구성됩니다. 각 트랙은 독립적으로 동작하며 신호를 개별적으로 처리합니다.

- **Audio Track:** 오디오 소스을 재생하는 트랙입니다.
- **MIDI Track:** 조명 효과나 MIDI 신호를 처리하는 트랙입니다.
- **Main(Master) Track:** 모든 개별 트랙의 신호가 최종적으로 모여 Main Track을 통해 스피커 또는 헤드폰으로 출력됩니다.

## 최적의 연주 환경을 위한 설정

안정적인 연주를 위해 하단의 안내를 참고하여 시스템 설정을 최적화하십시오.

### 오디오(Audio) 설정

1. **출력 장치 (Audio Output Device):** 현재 사용 중인 스피커나 헤드폰을 선택해주세요.
2. **버퍼 크기 (Buffer Size):** 지연 시간(레이턴시)을 최소화하기 위해 기본값인 4096에서 **1024 이하**로 조정하는 것을 권장합니다.

:::note[지연 시간 관리]
지연 시간(레이턴시)은 런치패드 버튼을 눌렀을 때 소리가 출력되기까지 걸리는 시간을 의미하며, 연주 경험에 큰 영향을 미칩니다. 전용 오디오 인터페이스를 사용 중이라면 **ASIO** 드라이버를 활용해 더 낮은 지연 시간(약 64-256 Samples)을 확보할 수 있습니다.
:::

### MIDI 장치 연동

Ableton Live가 런치패드 장치를 올바르게 인식하고 제어할 수 있도록 설정해야 합니다.

**Control Surface 및 입출력 포트 구성:**
환경 설정의 **Link/Tempo/MIDI** 탭에서 연결된 런치패드 모델을 선택하고, 모델별로 정의된 특정 포트를 할당합니다.

**MIDI Ports 활성화:**
하단의 MIDI Ports 리스트에서 런치패드로 전송되는 경로의 **Track**과 **Remote** 항목이 반드시 **On** 상태여야 합니다. 이는 라이트쇼 신호 전송과 맵핑 데이터 수신에 필수적인 요소입니다.
